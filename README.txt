Предметная область — библиотечная система (книги, авторы, читатели, выдачи).

База данных приведена к 3НФ: каждая таблица имеет первичный ключ, неключевые атрибуты зависят только от ключа, транзитивные зависимости отсутствуют. Связь многие-ко-многим реализована через таблицу book_author.

Полнотекстовый поиск реализован с помощью поля search_vector (tsvector), формируемого по полям title и annotation с использованием конфигурации russian, что обеспечивает учет морфологии русского языка. Для ранжирования используется функция ts_rank.

Для поиска по частичному совпадению используется расширение pg_trgm и GIN-индексы gin_trgm_ops для полей title и annotation. Это позволяет выполнять быстрый нечеткий поиск и поиск по части слова.

Добавлены индексы для внешних ключей и полей, используемых в WHERE, JOIN и ORDER BY. Сгенерировано более 5000 записей для тестирования.

Проведен анализ планов выполнения запросов с использованием EXPLAIN (ANALYZE, BUFFERS). В полнотекстовом поиске используется GIN-индекс (Bitmap Index Scan + Bitmap Heap Scan), что позволяет быстро находить подходящие записи и сортировать их по релевантности.

В запросе с ILIKE 'рефакторинг%' используется последовательное сканирование (Seq Scan): PostgreSQL проверяет все строки таблицы (около 5000), так как совпадений не найдено и использование индекса в данном случае менее эффективно. Также выполняется операция Sort для сортировки результата.

В триграммном поиске (оператор %) используется GIN trigram-индекс, что позволяет избежать полного сканирования таблицы и ускоряет поиск похожих строк.

В результате разработана БД, обеспечивающая эффективный и релевантный поиск и соответствующая требованиям OLTP.
